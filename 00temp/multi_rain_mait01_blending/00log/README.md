# 00log — 算法整理过程日志

记录**每一次原始算法整理**的步骤、目录映射、验证结论，以及**仍需人工补充**的项。

## 命名约定

```
<算法代号>_整理_<YYYYMMDD>.log
```

一次整理过程对应一个 log 文件；若同日多次整理，可在文件名后加序号，如 `_整理_20260723_02.log`。

## 日志建议结构

1. **基本信息**：算法代号、整理日期、整理人、原始代码来源  
2. **目录映射**：原路径 → 仓库标准路径  
3. **已完成项**：模块迁移、CLI 接入、单元测试等  
4. **待人工补充**：缺数据、缺配置、文档未对齐、环境依赖未锁定等  
5. **验证记录**：已跑通的命令、未跑通的原因
