# multi_rain_mait24_blending

## 算法概述

`multi_rain_mait24_blending` 用于逐日 24 小时累积降水的多源自适应集成。原始算法以 MAIT24 业务流程为核心，在站点维度计算多模式动态权重，进行降水频率匹配订正，再将集成站点产品插值为规则格点产品并输出业务格式。

核心能力包括：

- 读取历史多模式站点预报、当前多模式站点预报、实况站点数据和背景格点场。
- 按空间分区计算多降水量级 TS 评分。
- 使用 beta 历史评分记忆进行滚动自适应权重更新。
- 对当前多模式降水进行线性集成和频率匹配订正。
- 将集成站点降水与背景场融合，经过 Cressman 插值和平滑生成格点产品。
- 支持 Micaps3、Micaps4、NetCDF 业务产品写出。

## 算法分类

- 分类：`05blending`
- 分类依据：算法面向多源/多模式降水预报融合与订正，属于多模式融合算法。

## 处理流程

1. 解析 `time_inputs`、预报时效列表、运行参数和 ini 配置。
2. 读取站点表、模式路径、实况路径、输出路径、背景场路径和掩码文件。
3. 对每个起报时刻、每个预报时效读取历史模式、当前模式、实况和背景格点场。
4. 按空间分区裁剪训练区，计算多阈值 TS 评分与模式间相似性。
5. 融合历史 beta 评分，得到当前分区的模式权重。
6. 对当前模式降水进行线性加权，并进行频率匹配订正。
7. 写出站点产品和 beta 评分。
8. 将站点集成结果与背景约束合并，进行 Cressman 插值、平滑、频率匹配和长时效折减。
9. 写出最终格点产品。
