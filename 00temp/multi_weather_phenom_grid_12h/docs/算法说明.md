# 网格天气现象综合电码生成算法说明

**系统名称**：网格天气现象综合电码生成算法  
**标准依据**：QX/T 740-2024《基于网格预报的城镇预报生成规范 天气现象》  
**架构规范**：国省统筹算法技术规范（三级架构）  
**版本**：v1.0.0

---

## 一、算法应用场景

### 1.1 业务场景

城镇天气预报制作中，需要将网格数值预报（SCMOC 等）自动转换为**标准化天气现象综合电码**，用于：

| 场景 | 说明 |
|------|------|
| 城镇预报产品生成 | 由网格预报自动生成 12h 时段天气现象表述（如「阴转小雨」「晴间多云」） |
| 预报业务流水线 | 批量处理起报时次（00/12 UTC），覆盖 003~240h，共 20 个 12h 时段 |
| 质控与检验 | 输出 meteva 标准 6D NetCDF，可直接接入检验、对比、统计流程 |
| 替代人工判识 | 按行业标准统一判识/选取/逻辑关系规则，保证产品规范性与可复现性 |

### 1.2 输入与输出边界

- **输入**：逐 3 小时网格预报 NC（降水量、相态、云量、雾/霾/沙尘、雷暴、冰雹等）
- **输出**：每个 12h 时段一张网格场，格点值为 **5 位整型综合电码**（`KAABB`）
- **适用网格**：任意 `lat×lon` 规则网格；算法层只处理内存中的 `ndarray`，与具体投影无关

### 1.3 不适用 / 需注意

- 72h 后 FOG/HAZE/SAND/THUNDER/HAIL 常缺失：系统自动填零，**不影响**降水类与天空类判识
- 本算法生成的是「综合电码网格」，不是自然语言文本；表述需再用 `decode()` 解码

## 二、算法原理

依据标准附录 C 流程图，对每个 12h 时段、每个格点执行四步诊断：

```
网格预报变量
    │
    ▼
① 判识（附录A）  →  31 种天气现象是否在 12h / 各 3h 时步出现
    │
    ▼
② 选取（第4章）  →  影响级最高的现象 A + 可独立出现的现象 B（或仅 A）
    │
    ▼
③ 逻辑关系（第5章/附录E）→  单一 / 转 / 间 / 伴有，并确定表述先后
    │
    ▼
④ 编码（第6章）  →  5 位综合电码 KAABB
```

### 2.1 判识原理（附录 A）

对每个格点，结合 **12h 累积规则** 与 **精细 3h 辅助规则** 判识 31 种现象：

| 类别 | 依据 | 判识要点 |
|------|------|----------|
| 降水（表 A.1） | R03 + PTYPE03（+ THUNDER/HAIL） | 12h 分相态累积量分级；雷阵雨/冰雹/雨夹雪/冻雨另有组合条件 |
| 视程障碍（表 A.2） | FOG / HAZE / SAND | 等级场直接映射到雾/霾/沙尘电码 |
| 天空状况（表 A.3） | TCC | 12h 内至少 1 个 3h 时步满足云量阈值即判为出现 |

每种现象输出：

- `12h`：该 12h 窗口内是否出现（用于选取 A/B）
- `fine`：4 个 3h 时步是否出现（用于独立出现判断、交替、「转」的先后）

### 2.2 选取原理（第 4 章）

1. **现象 A**：在 12h 内已出现的现象中，取**影响级最高**者（影响级数字越小优先级越高）
2. **现象 B**：在其余已出现现象中，取影响级次高、且与 A **非互斥**、且在精细时步上存在**独立出现时间**者
3. 无合适 B → 仅保留 A（后续逻辑为「单一」）
4. 互斥组（附录 D）：降雨类 / 降雪类 / 雾类 / 霾类 / 沙尘类——组内只保留影响级最高的一种

### 2.3 逻辑关系原理（第 5 章 / 附录 E）

| 关系 | 电码 K | 条件概要 |
|------|--------|----------|
| 单一 | 0 | 无 B，或仅一种有效现象 |
| 伴有 | 3 | A、B 分别为雾类与霾类 |
| 间 | 2 | A/B 为晴–多云–阴–小雨的特定候选对，且在精细时步上**交替出现** |
| 转 | 1 | 其余双现象情况；按首次出现时间确定先后，相同时按持续时长/影响级破平 |

### 2.4 编码原理（第 6 章）

```
KAABB = K × 10000 + AA × 100 + BB
```

| 字段 | 含义 |
|------|------|
| K | 逻辑关系：0 单一 / 1 转 / 2 间 / 3 伴有 |
| AA | 表述在前的现象电码（00~65 等，两位） |
| BB | 表述在后的现象电码；单一时 AA=BB |

标准示例：

| 综合电码 | 预报表述 |
|----------|----------|
| 00101 | 多云 |
| 10207 | 阴转小雨 |
| 20001 | 晴间多云 |
| 35661 | 中度霾伴有轻雾 |

## 三、算法原理的实现方式

### 3.1 三级架构与职责边界

```
Level 3  算法应用           src/main.py（process）+ cli/__main__.py 路由
Level 2  算法插件 (Plugins)  DIA_WeatherPhenomIdentifier / Selector / LogicJudger / Encoder
Level 1  基础函数 / 工具     identify / select / judge / encode；src/utils/data_loader、output
```

- **src/main.py**：统一执行入口（命令行 + `process`）；编排并行并调用数据加载与计算
- **cli**：`python -m cli` 转发主程序
- **src/utils/data_loader**：读输入 NC；插件层只接收内存数据

### 3.2 四步 Plugin 实现映射

| 步骤 | Plugin | 核心函数 | 实现要点 |
|------|--------|----------|----------|
| 判识 | `DIA_WeatherPhenomIdentifier` | `identify()` | NumPy 向量化：分相态累加、阈值掩膜、等级映射；TCC 自动识别 0~1 / 0~100 |
| 选取 | `DIA_WeatherPhenomSelector` | `select()` | 现象用 **int8 索引**；预计算互斥矩阵 `_EXCL`；按影响级序取 A，再扫非互斥 B 且要求 `fine_B & ~fine_A` |
| 逻辑 | `DIA_WeatherPhenomLogicJudger` | `judge()` | 预计算伴有矩阵 `_ACCOMPANY`、间候选 `_JIAN_CAND`；计算首次出现/持续时长/是否交替 |
| 编码 | `DIA_WeatherPhenomEncoder` | `encode()` | 索引→电码 LUT `_CODE_INT_LUT`，整网一次向量化：`K*10000+AA*100+BB` |

### 3.3 端到端数据流

```
src.utils.data_loader.load_segment()
    → data_dict: {变量: ndarray[4, lat, lon]}
src.processor.run_segment()
    → Identifier.process → occur
    → Selector.process   → idx_A, idx_B          (int8)
    → LogicJudger.process→ logic, idx_fa, idx_fb (int8)
    → Encoder.process    → result                (int32, lat×lon)
    → _save_nc()         → meteva 6D NetCDF
```

`processor.py` 内四个 Plugin 为**模块级单例**，避免重复预计算矩阵。

### 3.4 性能相关实现

| 手段 | 作用 |
|------|------|
| int8 索引全程传递 | 避免 `dtype=object` 字符串比较 |
| 互斥/伴有/间候选预计算矩阵 | 模块 import 时一次性构建 |
| 时段间 + 文件级线程池 | `src/main.py`：`max_seg_workers` × `max_workers` |
| NumExpr 线程上限 | `--numexpr-threads`，降低高并发 CPU 争用 |

验证脚本见 `nbs/run_bench.py`（分阶段计时）、`nbs/run_test.ipynb`（Plugin/集成/pytest）。

## 四、算法参数说明

业务阈值与规则集中在 `resource/weather_config.py`；结构常量在 `resource/data_schema.py`；运行环境在 `resource/config.py`。

### 4.1 输入变量参数

| 变量 | 含义 | 单位/编码 | 有效时效 |
|------|------|-----------|----------|
| R03 | 3h 累计降水量 | mm | 003~240h |
| PTYPE03 | 降水相态 | 0 无 / 1 雨 / 2 雪 / 3 雨夹雪 / 4 冻雨 | 003~240h |
| TCC | 总云量 | % 或 0~1（`max≤1.1` 时自动 ×100） | 003~240h |
| FOG | 雾等级 | 0~5（轻雾→特强浓雾） | 003~072h |
| HAZE | 霾等级 | 0~4（轻度→严重） | 003~072h |
| SAND | 沙尘等级 | 0~3（扬沙/浮尘→强沙尘暴） | 003~072h |
| THUNDER | 雷暴标志 | ≥1 有雷暴 | 003~072h |
| HAIL | 冰雹标志 | ≥1 有冰雹 | 003~072h |

单时段内存形状约定：`ndarray[4, lat, lon]`（4 个 3h 时步）。

### 4.2 判识阈值（附录 A）

**降雨 12h 累积（mm）**

| 电码 | 现象 | 区间 |
|------|------|------|
| 07 | 小雨 | [0.1, 10) |
| 08 | 中雨 | [10, 25) |
| 09 | 大雨 | [25, 50) |
| 10 | 暴雨 | [50, 100) |
| 11 | 大暴雨 | [100, 250) |
| 12 | 特大暴雨 | ≥250 |

**降雪 12h 累积（mm 液态当量）**

| 电码 | 现象 | 区间 |
|------|------|------|
| 14 | 小雪 | [0.1, 2.5) |
| 15 | 中雪 | [2.5, 5) |
| 16 | 大雪 | [5, 10) |
| 17 | 暴雪 | [10, 20) |
| 36 | 大暴雪 | [20, 30) |
| 37 | 特大暴雪 | ≥30 |

**组合降水**

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `THUNDER_RAIN_THRESH` | 0.1 mm | 雷阵雨：总降水阈值 + 有雷暴 |
| `HAIL_RAIN_THRESH` | 0.1 mm | 雷阵雨伴冰雹：总降水 + 降水时段有冰雹 |
| `SLEET_THRESH` | 0.1 mm | 雨夹雪：有时段相态为雨夹雪且降水达标 |
| `FRZRAIN_THRESH` | 0.1 mm | 冻雨：总降水达标且有冻雨相态 |
| `FINE_PRECIP_THRESH` | 0.0 mm | 精细时步：降水量 > 0 视为有降水 |

**天空状况（TCC，%）**

| 电码 | 现象 | 条件 |
|------|------|------|
| 00 | 晴 | cl ≤ 30 |
| 01 | 多云 | 30 < cl < 90 |
| 02 | 阴 | cl ≥ 90 |

**等级→电码映射**

| 要素 | 映射 |
|------|------|
| FOG 1~5 | 61~65（轻雾→特强浓雾） |
| HAZE 1~4 | 55~58（轻度→严重霾） |
| SAND 1~3 | 30 / 20 / 31（扬沙或浮尘 / 沙尘暴 / 强沙尘暴） |

### 4.3 影响级与互斥（附录 B / D）

影响级数字越小优先级越高（特大暴雨=1 … 晴=31）。互斥组：

- 降雨类：12, 11, 10, 05, 09, 19, 08, 04, 07  
- 降雪类：37, 36, 17, 16, 15, 14  
- 雾类：65, 64, 63, 62, 61  
- 霾类：58, 57, 56, 55  
- 沙尘类：31, 20, 30  

雨夹雪（06）不属互斥组，可独立参与选取。

### 4.4 逻辑关系参数（附录 E）

| 参数 | 说明 |
|------|------|
| `FOG_CODES` / `HAZE_CODES` | 构成「伴有」的雾、霾电码集合 |
| `JIAN_CANDIDATE_PAIRS` | 「间」候选对：（晴,多云）（多云,阴）（阴,小雨）及对称对 |
| `LOGIC_RELATION_CODE` | 单一=0，转=1，间=2，伴有=3 |

### 4.5 运行 / CLI 参数

| 参数 | 默认 | 说明 |
|------|------|------|
| `init_times` | 必填 | 起报时次 `YYYYMMDDHH`，可多个 |
| `--output-dir` | `./PHENOM` | 输出根目录 |
| `--seg-range START END` | 1~20 | 处理的 12h 时段范围（含端点） |
| `--data-root` | 环境变量 / config | 覆盖 `SCMOC_DATA_ROOT` |
| `--max-seg-workers` | 3 | 时段间并行数（推荐 3~5） |
| `--max-workers` | 4 | 单时段文件并发读（推荐 4~8） |
| `--numexpr-threads` | 不设 | 限制 NumExpr 占用核数 |
| `--stats-only` | off | 只算统计不写盘 |

数据根目录优先级：`--data-root` > 环境变量 `SCMOC_DATA_ROOT` > `resource/config.py` 默认值。

### 4.6 输出路径与字段

```
{output_dir}/{YYYYMMDDHH}/{YYYYMMDDHH}.{TTT}.nc
```

- `TTT = seg_idx × 12`（时段结束预报时效，如段 1 → `012`）
- 变量名：`phenom_code`
- 维度：`[member, level, time, dtime, lat, lon]`（meteva 6D）

## 五、算法调用

### 5.1 命令行（推荐生产）

```powershell
# 单起报，全部 20 段（003~240h）
python -m cli 2026030100
# 等价：python src/main.py 2026030100

# 多起报
python -m cli 2026030100 2026030112

# 只跑 0~72h（第 1~6 段）
python -m cli 2026030100 --seg-range 1 6

# 指定数据与输出，并调并行
python -m cli 2026030100 `
  --data-root \\server\share\SCMOC `
  --output-dir D:/output `
  --max-seg-workers 5 --max-workers 8

# 仅统计不落盘
python -m cli 2026030100 --stats-only
```

**退出码**：`0` 全部成功；`1` 数据缺失（非致命）；`2` 计算错误（致命）。

### 5.2 Python 主程序（`src.main.process`）

```python
from src.main import process

process(
    "2026030100",
    output_dir="./PHENOM",
    seg_range=(1, 6),       # 可选
    max_seg_workers=5,
    max_workers=8,
)
```

### 5.3 单时段：加载 + 计算

```python
from src.utils.data_loader import load_segment
from src.processor import run_segment

data_dict, shape, lat_arr, lon_arr = load_segment("2026030100", seg_idx=1)
result = run_segment(
    data_dict, shape, lat_arr, lon_arr,
    init_time="2026030100", seg_idx=1,
    output_dir="./PHENOM",
)
# result: ndarray[lat, lon] int32
```

### 5.4 Plugin API（纯内存，便于嵌入与单测）

```python
from src.identifier import DIA_WeatherPhenomIdentifier
from src.selector import DIA_WeatherPhenomSelector
from src.logic_judger import DIA_WeatherPhenomLogicJudger
from src.encoder import DIA_WeatherPhenomEncoder, decode

identifier = DIA_WeatherPhenomIdentifier()
selector   = DIA_WeatherPhenomSelector()
judger     = DIA_WeatherPhenomLogicJudger()
encoder    = DIA_WeatherPhenomEncoder()

# data_dict 已由调用方准备好：{变量: ndarray[4,lat,lon]}
occur                 = identifier.process(data_dict)
idx_A, idx_B          = selector.process(occur)
logic, idx_fa, idx_fb = judger.process(idx_A, idx_B, occur)
result                = encoder.process(idx_fa, idx_fb, logic)

# 单点解读
print(decode(int(result[100, 200])))
# → {'预报表述': '阴转小雨', '综合电码': '10207', ...}
```

等价函数式接口：`identify` / `select` / `judge` / `encode`（见 `src/` 各模块）。
