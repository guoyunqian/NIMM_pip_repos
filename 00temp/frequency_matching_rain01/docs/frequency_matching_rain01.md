# 逐小时降水频率匹配订正算法说明

## 算法概述

逐小时降水频率匹配订正算法用于单一数值模式 1 小时降水预报的统计后处理订正。算法基于历史同期模式-实况配对样本，在空间分块内进行相似个例筛选、光流位移订正和频率匹配强度订正，输出订正后的 MICAPS3 站点场和 MICAPS4 格点场。

## 核心能力

- 读取单模式逐 1 小时降水预报和 MICAPS3 实况资料。
- 基于 4 年历史同期样本构建模式-实况配对样本库。
- 按 TS 和 BIAS 综合评分筛选相似历史个例。
- 使用光流和半拉格朗日方法订正雨区位置偏差。
- 使用 CDF 分位数映射进行频率匹配强度订正。
- 通过 Cressman 插值和站点约束生成格点订正产品。

## 目录结构

| 路径 | 内容 |
| --- | --- |
| `src/runner.py` | 主业务流程入口 |
| `src/proc/` | 频率匹配、相似筛选、光流、外推和空间插值算法 |
| `src/data/` | MICAPS、二进制格点和站点数据结构及读写 |
| `src/verify.py` | 验证脚本 |
| `src/verify_data/` | 原始验证样例数据 |
| `cli/` | 命令行入口 |
| `resource/` | 路径配置、网格配置、站点表和掩膜资源 |
| `test/` | 单元测试和对比测试脚本 |
| `test_data/` | 测试数据说明 |
| `nbs/` | notebook 示例 |
| `utils/` | 路径、日志、字符串模板和并行工具 |

## 使用入口

默认运行：

```bash
python -m cli
```

指定模式和时刻：

```bash
python -m cli ecmwf 202604081130
python -m cli ecmwf 202604081130 202604081800
```

直接调用主程序：

```bash
python src/runner.py ecmwf 202604081130
```

## 关键配置

- `resource/qpf_fm.ini`：索引日志、路径配置、网格配置、站点表和掩膜文件。
- `resource/path.json`：配置不同模式的预报、实况和输出路径模板。
- `resource/config.json`：配置订正区域、网格分辨率、分块边界和掩膜参数。
- `resource/sta.info`：订正站点列表。
- `resource/mask010.dat`：有效订正区域掩膜。

## 注意事项

1. 当前整理阶段保持原始导入方式，代码仍使用 `src`、`proc`、`data` 和 `utils` 直接导入。
2. 默认配置文件中保留生产业务路径，离线运行前需要替换为本地样例路径。
3. 原始代码未提供标准插件类和 `process` 方法，正式入库前可按仓库规范补充插件封装。
4. `src/verify_data/` 中包含验证样例数据，但完整业务流程仍依赖模式、实况、站点表、掩膜和输出目录等外部环境。
